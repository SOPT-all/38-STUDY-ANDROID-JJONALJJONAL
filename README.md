```

```

## ⚙️ 코드 제출 및 관리 가이드
프로그래머스 문제를 해결한 후, 아래 규칙에 맞춰 GitHub에 PR을 남겨주세요.

1. 파일 및 폴더 구조
자신의 이름으로 폴더를 생성하여 관리합니다.

폴더명: 본인영문이름 (ex: seungjae-choi)

2. 브랜치 전략
주차별로 브랜치를 생성하여 작업 후 자신의 폴더에 병합(Merge)합니다.

브랜치명: week[주차]/본인영문이름 (ex: week1/seungjae-choi)

문제 풀이 파일명: 파일은 '레벨_문제_본인영문이름.kt'형태의 네이밍을 사용

PR 방식: 'week[주차]/본인영문이름'형태의 네이밍을 사용

코드리뷰를 받은 후 스터디 끝나고 -> main(또는 각자 폴더)으로 Merge
